# 怪兽设计工作室 B2B 网站定位改造方案

日期：2026-07-11
状态：方向已批准，待书面方案审核

## 一、改造目标

将现有的综合型室内设计作品集，重新定位为面向海外室内设计事务所、建筑师及商业装修承包商的远程生产与技术交付服务网站。

网站首先展示的服务为 CAD 制图、施工图、图纸深化和技术文档支持；室内效果图作为第二主服务；设计表现、FF&E 和材料方向支持继续保留，但不再作为首页的主要销售重点。

## 二、为什么选择这个方向

本次共比较了三种定位方案：

1. **国际化全案室内设计工作室。** 这种方案可以保留目前的表达方式，但服务范围过宽，需要同时说服多类客户，也会直接与能够提供现场服务、报批和本地协调的成熟设计公司竞争。
2. **效果图优先的可视化工作室。** 这种方向视觉吸引力强，现有作品也能支持，但市场同质化严重，容易进入价格竞争，同时浪费了网站中最有差异化的技术图纸内容。
3. **技术交付为主、效果图为辅的远程生产伙伴。** 这是最终选择的方案。它能让海外专业团队快速理解为什么需要怪兽设计工作室：当内部人员繁忙、临近交付节点或暂时缺少生产能力时，可获得灵活的远程图纸与可视化支持。

## 三、目标客户

### 主要客户公司

- 中小型室内设计事务所
- 需要室内生产支持的建筑事务所
- 商业装修和室内精装承包商
- 临时需要增加远程交付能力的项目团队

### 主要决策人

- 工作室创始人和设计总监
- 项目建筑师和项目经理
- 技术设计负责人
- 装修承包商的项目经理和估算负责人

### 主要需求场景

- 临近项目节点时，内部团队工作量过大
- 项目需要根据已有设计资料进行图纸深化
- 提案或客户沟通需要高质量的室内效果图
- 团队希望先通过一个小型付费试单验证合作能力

### 不作为主要目标的客户

- 需要本地全案设计服务的私人住宅业主
- 需要本地报批盖章或持证工程师认证的客户
- 需要当地现场管理，但工作室在该市场没有本地团队的客户

## 四、定位陈述

怪兽设计工作室为海外设计事务所、建筑师和商业装修承包商提供远程 CAD 制图、技术文档、图纸深化及室内效果图支持。

## 五、信息传达顺序

1. **服务对象：** 海外设计事务所、建筑师和商业装修承包商。
2. **解决的问题：** 生产能力不足、交付期限压力，以及临时缺少技术或可视化人员。
3. **第一主服务：** 根据客户提供的资料和已确认范围，提供 CAD 制图与技术文档支持。
4. **第二主服务：** 用于设计深化、提案和客户沟通的室内效果图。
5. **合作方式：** 先审核项目范围，再从小型付费试单或范围明确的项目包开始。
6. **信任边界：** 保密处理、明确交付物、集中反馈，不暗示具备当地专业认证或报批资质。

## 六、建议的网站结构

网站继续采用静态单页作品集形式，保留现有视觉风格和项目图片。

1. **首屏**
   - 直接说明目标客户以及技术交付价值。
   - 主按钮：发送图纸或项目需求，申请范围审核。
   - 次按钮：查看技术图纸和效果图作品。

2. **我们服务的团队**
   - 室内设计事务所
   - 建筑师和项目团队
   - 商业装修承包商

3. **我们帮助解决的问题**
   - 项目高峰期的生产能力不足
   - 临近交付节点的图纸支持
   - 提案和客户沟通所需的可视化支持
   - 范围明确的远程交付

4. **核心服务**
   - CAD 制图与技术文档
   - 室内效果图
   - 设计表现与 FF&E 支持

5. **技术图纸作品**
   - 将现有图纸案例提前到综合项目作品之前。
   - 说明服务范围、交付内容和能力边界，不暗示工程认证。

6. **效果图作品**
   - 保留商业、酒店、办公、别墅和住宅中的优质图片。
   - 将图片重新描述为对设计深化、提案和客户沟通的生产支持。

7. **交付内容与合作方式**
   - 说明文件格式、源资料、节点、修改轮次和最终交付物按项目确认。
   - 不列出未经确认的软件能力或保证性工期。

8. **合作流程**
   - 提供图纸和项目需求
   - 确认范围、交付物、周期和报价
   - 适合时先进行小型付费试单
   - 根据集中整理的意见进行审核修改
   - 交付最终确认文件

9. **关于我们**
   - 将工作室描述为远程生产伙伴，而不是泛化的国际全案设计公司。

10. **项目询盘**
    - 引导客户提供项目类型、地点、面积、现有图纸、所需服务、文件格式、目标日期和预算范围。
    - 第一版继续使用邮箱和 WhatsApp，不接入第三方表单服务。

## 七、逐个文件的修改范围

### `index.html`

- 重写页面标题、搜索描述、社交分享文案、首屏、服务标题、客户类型、合作流程、工作室介绍和询盘文案。
- 将技术图纸内容移动到综合效果图作品之前。
- 删除“所有项目类型都能支持”之类过宽的说法，改成现有作品能够证明的具体能力。
- 将行动按钮改为“申请项目范围审核”和“查看作品”。
- 增加简短的保密、交付和服务边界说明。
- 仅使用网站已有的公开联系方式和服务信息，添加真实的 `ProfessionalService` 结构化数据。

### `assets/data.js`

- 重写项目摘要和服务标签，说明每个项目在专业生产流程中的作用。
- 保留当前项目地点和保密表述。
- 不把概念效果图描述为已经建成的实际项目。
- 保持现有图片路径、尺寸和图库行为不变。

### `script.js`

- 保留现有弹窗、筛选、导航和无障碍行为。
- 只修改新内容结构确实需要调整的选择器或筛选值。
- 继续通过安全的文本节点插入项目文字，不使用未经处理的原始 HTML。

### `styles.css`

- 延用当前设计系统。
- 只进行新板块顺序、服务层级和移动端阅读所需的布局调整。
- 不增加前端框架或外部依赖。

### `privacy-policy.html`

- 第一版继续保持“不安装追踪工具”的现状。
- 只有当新页面结构或联系方式说明确实需要时，才调整相关文字。
- 未来如果安装统计、广告代码或第三方表单，必须先重新审核隐私政策。

### `sitemap.xml` 和 `robots.txt`

- 在实际购买独立域名前，继续使用当前 GitHub Pages 地址。
- 不提前写入不存在的域名。

### `.agents/product-marketing.md`

- 保存已经确认的目标客户、定位、服务顺序、宣传边界和主要转化动作，供后续营销工作统一使用。

## 八、客户转化路径

第一版使用简短、低阻力的询盘路径：

1. 潜在客户通过主动开发、推荐或搜索进入网站。
2. 首屏让客户确认该服务专门面向专业项目团队。
3. 技术图纸和效果图案例证明生产能力。
4. 合作流程降低客户对远程协作的不确定感。
5. 客户通过邮件或 WhatsApp 发送需求和已有图纸。
6. 怪兽设计工作室审核资料，提出付费试单或范围明确的项目方案。

网站浏览或点击案例不算合格线索。合格询盘应包含真实项目需求、所需服务和预计时间。

## 九、真实性与风险控制

- 不虚构客户、评价、业绩数据、资质、软件能力、服务地区或交付速度。
- 未经明确许可，不公开客户名称、图签、源图纸或保密标识。
- 不暗示图纸服务包含当地规范审批、工程师认证或报批盖章。
- 原始项目文件不得进入公开仓库及其 Git 历史。
- 公开仓库历史清理属于单独的安全任务，因为它涉及破坏性历史重写和协调后的强制推送。

## 十、无障碍与异常情况处理

- 保留键盘操作、弹窗焦点恢复、移动端导航、替代文本和减少动画支持。
- 即使 JavaScript 加载失败，联系方式仍必须可见。
- 所有新增板块在没有动画时也必须能够正常阅读。
- 即使图库数据无法加载，页面仍应说明服务内容并显示可用联系方式。

## 十一、搜索优化范围

- 页面文字聚焦远程 CAD 制图、室内技术文档、施工图支持和室内效果图支持。
- 避免机械重复关键词。
- 站点地图中继续保留一个主页面和现有隐私政策页面。
- 结构化数据只使用公开网站中可以证实的事实。
- 独立域名、网站统计、广告代码和广告专用落地页不属于第一版范围。

## 十二、验证要求

推送网站改动前必须完成：

- 验证 JavaScript 语法。
- 检查所有本地图片、样式、脚本、导航链接、邮箱链接、WhatsApp 和社交链接。
- 确认没有引入非预期的外部脚本、iframe、密钥或混淆代码。
- 对照已批准的定位和宣传边界检查全部新文案。
- 测试桌面端和移动端布局。
- 测试键盘导航和项目弹窗。
- 确认部署后的 GitHub Pages 文本文件与批准的 GitHub 提交一致。

## 十三、完成标准

满足以下条件时，本次改造才算完成：

- 首次访问者能在第一屏看懂服务对象、第一主服务和下一步动作。
- 技术文档明显是第一主服务，效果图是第二主服务。
- 网站不再把怪兽设计工作室描述为不受范围限制的国际全案设计公司。
- 现有项目图库和联系方式继续正常工作。
- 没有加入无法证明的说法或保密源资料。

## 十四、第一版不包含的内容

- 购买或配置独立域名
- 付费广告投放
- Google Analytics、Google Ads、Meta Pixel 或用户同意管理工具
- 内容管理系统、数据库、客户门户或第三方询盘表单
- 公布固定价格或保证性交付时间
- 破坏性的 Git 历史清理
